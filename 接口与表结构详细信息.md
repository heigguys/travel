# 公司旅行管理系统接口与表结构详细信息

本文档根据当前项目源码整理，覆盖后端 Controller、DTO、Service、Mapper、`schema.sql` 以及前端 `app.js` 中实际调用的接口。

## 1. 项目概览

| 项目项 | 说明 |
| --- | --- |
| 后端目录 | `back-end` |
| 前端目录 | `front-end` |
| 后端框架 | Spring Boot `3.5.14` |
| Java 版本 | Java `17` |
| ORM / SQL 框架 | MyBatis |
| 数据库 | MySQL |
| 后端默认端口 | `8080` |
| API Base URL | `http://localhost:8080/api` |
| 前端调用方式 | `fetch(API_BASE + path, { credentials: "include" })` |
| 数据库名 | `db_travel` |
| 初始化 SQL | `back-end/src/main/resources/schema.sql`、`data.sql` |
| 文件存储根目录 | `storage`，由配置项 `app.storage.root=storage` 指定 |

### 1.1 CORS 配置

后端对 `/api/**` 开启跨域，并允许携带 Cookie。

| 配置项 | 值 |
| --- | --- |
| 允许方法 | `GET`、`POST`、`PUT`、`DELETE`、`OPTIONS` |
| 允许 Header | `*` |
| 允许凭证 | `true` |
| 允许来源 | `http://localhost:8081`、`http://localhost:8082`、`http://127.0.0.1:8081`、`http://127.0.0.1:8082` |

## 2. 通用规则

### 2.1 登录与 Session

后端使用 `HttpSession` 保存登录状态，登录成功后在 Session 中写入：

| Session Key | 说明 |
| --- | --- |
| `USER_ID` | 当前登录用户 ID |

除 `POST /api/auth/login` 外，其余业务接口都会通过 `AuthService.currentUser(session)` 校验登录状态。未登录或登录用户不存在时返回业务失败。

### 2.2 用户角色

| 角色 | 说明 |
| --- | --- |
| `ADMIN` | 管理员，可新增、编辑、删除旅行计划，可查看所有旅行计划和所有咨询 |
| `USER` | 普通员工，只能查看已公开旅行计划，只能操作自己的申请和随行人 |

### 2.3 统一 JSON 响应格式

除 PDF/附件下载接口外，接口统一返回 `ApiResponse<T>`。

```json
{
  "success": true,
  "message": "ok",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `success` | `boolean` | 是否成功 |
| `message` | `string` | 响应消息；成功数据接口通常为 `ok` |
| `data` | `T` / `null` | 实际返回数据；纯消息接口为 `null` |

### 2.4 通用错误响应

| 场景 | HTTP 状态码 | 响应格式 | message 来源 |
| --- | --- | --- | --- |
| 业务异常 | `400` | `ApiResponse<Void>` | Service 抛出的业务消息 |
| 参数校验失败 | `400` | `ApiResponse<Void>` | `输入内容不完整或格式不正确` |
| 其他异常 | `500` | `ApiResponse<Void>` | `服务器处理失败：` + 异常信息 |

示例：

```json
{
  "success": false,
  "message": "请先登录",
  "data": null
}
```

### 2.5 非 JSON 返回接口

| URL | 返回类型 | Content-Type | 说明 |
| --- | --- | --- | --- |
| `GET /api/plans/{id}/file` | PDF 附件流 | `application/pdf` | 旅行计划附件，浏览器内联打开 |
| `GET /api/my-applications/export.pdf` | PDF 文件字节 | `application/pdf` | 我的申请导出 PDF，浏览器下载 |

## 3. 接口详细信息

### 3.1 认证接口

#### 3.1.1 登录

| 项目 | 说明 |
| --- | --- |
| URL | `/api/auth/login` |
| Method | `POST` |
| 权限 | 无需登录 |
| Content-Type | `application/json` |
| Controller | `AuthController.login` |

请求 Body：

| 参数 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- |
| `employeeNo` | `string` | 是 | `@NotBlank` | 员工编号 |
| `password` | `string` | 是 | `@NotBlank` | 明文密码，后端用 MD5 比对 |

请求示例：

```json
{
  "employeeNo": "A001",
  "password": "123456"
}
```

返回 `data`：`User`

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | `number` | 用户 ID |
| `employeeNo` | `string` | 员工编号 |
| `name` | `string` | 姓名 |
| `email` | `string` | 邮箱 |
| `role` | `ADMIN` / `USER` | 角色 |
| `passwordMd5` | `null` | 登录返回前会清空 |
| `enabled` | `boolean` | 是否启用 |

业务规则：

| 规则 | 说明 |
| --- | --- |
| 账号查找 | 只查询 `enabled = true` 的用户 |
| 密码校验 | 使用 `Md5Util.md5(password)` 与 `users.password_md5` 比对 |
| 登录成功 | Session 写入 `USER_ID` |
| 登录失败 | 返回 `员工编号或密码错误` |

#### 3.1.2 退出登录

| 项目 | 说明 |
| --- | --- |
| URL | `/api/auth/logout` |
| Method | `POST` |
| 权限 | 需登录 |
| Controller | `AuthController.logout` |

请求参数：无。

返回：

```json
{
  "success": true,
  "message": "已退出登录",
  "data": null
}
```

业务规则：调用 `session.invalidate()` 清除当前会话。

#### 3.1.3 获取当前用户

| 项目 | 说明 |
| --- | --- |
| URL | `/api/auth/me` |
| Method | `GET` |
| 权限 | 需登录 |
| Controller | `AuthController.me` |

请求参数：无。

返回 `data`：`User`，字段同登录接口，`passwordMd5` 返回前置为 `null`。

常见错误：

| message | 说明 |
| --- | --- |
| `请先登录` | Session 中没有 `USER_ID` |
| `登录状态已失效` | Session 中用户 ID 无法查询到启用用户 |

| 项目 | 说明 |

| 更新方式 | 将新密码 MD5 写入 `users.password_md5` |

#### 3.3.1 申请旅行计划

| 项目 | 说明 |

| Method | `POST` |
| 权限 | 需登录 |
| Content-Type | `application/json` |
| Controller | `ApplicationController.apply` |

Path 参数：

| 参数 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `planId` | `number` | 是 | 旅行计划 ID |

请求 Body：

| 参数 | 类型 | 必填 | 校验 | 说明 |
| --- | --- | --- | --- | --- |
| `applicantCount` | `integer` | 是 | `@NotNull`、`@Min(1)` | 申请人数 |
| `optionText` | `string` | 否 | 最大长度由表字段限制为 `500` | 选项/备注 |

请求示例：

```json
{
  "applicantCount": 3,
  "optionText": "希望安排靠窗座位"
}
```

业务规则：

| 规则 | 说明 |
| --- | --- |
| 计划校验 | 计划不存在或未公开时返回 `旅行计划不存在或未公开` |
| 容量校验 | `当前 ACTIVE 总人数 - 本人旧申请人数 + 新申请人数` 不能超过计划定员 |
| 重复申请 | 同一用户同一计划已有 `ACTIVE` 申请时，更新原申请记录而非新增 |
| 新申请状态 | `ACTIVE` |
